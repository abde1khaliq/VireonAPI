Welcome to Vireon API

Hello there! Welcome to the Vireon API documentation. We're excited to help you integrate our advanced content moderation capabilities for text and images into your applications. Let's get you started on creating safer digital spaces.

First Things First: Get Your API Key

Before you can make any API calls, you'll need an API key. Don't worry, it's quick and easy to get one!

  1. Visit your Dashboard
  2. Sign in or create an account if you haven't already
  3. Navigate to the API Keys section
  4. Generate a new API key

Pro tip: Give your key a descriptive name so you remember what it's for later!

GET /v1/analyze/content

Check the status of the Vireon API service. This endpoint returns the current status and version information.

Example Request

bash 
curl -X GET "https://api.vireon.com/v1/analyze/content" \
  -H "Authorization: Bearer YOUR_API_KEY"

Example Response

json

"violation": true,
"harm_type": ["profanity"],
"confidence": 1.0,
"severity": "moderate",

    }
}

Securing Your Requests: Authentication

All API requests require authentication using an API key. Think of this as your unique passport to access Vireon's capabilities. Include your API key in the Authorization header of every request.

Header Parameters

Parameter Type Description
Authorization string Bearer token with your API key: "Bearer YOUR_API_KEY"
Content-Type string Should be "application/json" for all requests

Example

bash 
curl -X POST "https://api.vireon.com/v1/analyze/content" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"user_input": "Your text to analyze"}'
Keep your API key secure! Never expose it in client-side code or public repositories. If you suspect your key has been compromised, regenerate it immediately from your dashboard.

Keeping Text Clean: Content Moderation

Analyze text content for profanity, hate speech, and other inappropriate content. Our AI understands context and nuance, detecting even subtle forms of harmful content across multiple languages.

POST /v1/analyze/content

Send us text content you'd like to analyze, and we'll return a detailed analysis with confidence scores for various categories of inappropriate material.

Request Parameters

Parameter Type Required Description
user_input string Yes The text content you want us to analyze

Example Request

bash 
curl -X POST "https://api.vireon.com/v1/analyze/content" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "user_input": "Your text content to analyze for inappropriate material",
  }'

Example Response

json 
{
"result": {
"violation": true,
"harm_type": ["profanity"],
"confidence": 1.0,
"severity": "moderate",
"reasoning": "The input contains a highly offensive expletive."
    }
}
Did you know? Our text moderation supports over 15 languages and can detect context-specific inappropriate content that simple word filters would miss.

Visual Content Safety: Image Moderation

Keep your platforms visually appropriate with our advanced image analysis. We detect inappropriate imagery including nudity, violence, and other graphic material, plus we perform OCR to catch text within images.

POST /v1/analyze/image

Send us an image URL or upload an image directly, and we'll analyze it for inappropriate content with detailed results.

Request Parameters

Parameter Type Required Description
image_url string Yes* URL of the image you want us to analyze (if not uploading directly)

Example Request

  -H "Authorization: Bearer YOUR_API_KEY" \

Managing API Usage: Rate Limits

To ensure fair usage and system stability for all our users, we've implemented rate limits based on your subscription plan. Let's walk through how this works.

What are rate limits?

Rate limits control how many requests you can make to our API within a specific time frame. This prevents abuse, ensures fair access for everyone, and keeps our system running smoothly.

How our rate limiting works

We use the refilling bucket algorithm to manage rate limits. Here's how it works:

Think of it like a water bucket:

  • Your plan determines your bucket size (request limit)
  • Each API request uses some water (tokens) from the bucket
  • Water slowly refills over time at a steady rate
  • If the bucket is empty, you'll need to wait for it to refill

Example scenario

If your plan allows 1,000 requests per hour:

  • Your bucket starts with 1,000 tokens
  • Each API call uses one token
  • Tokens refill gradually throughout time
  • This lets you handle both steady usage and short bursts

What happens if you exceed limits?

If you run out of tokens, our API will respond with a 429 Too Many Requests status code. You'll need to wait for tokens to refill before making more requests.

Checking your usage

You can always monitor your current usage in your dashboard. We recommend setting up monitoring to avoid unexpected rate limits in your application.

Need higher limits? Check out our subscription plans to find one that matches your usage needs. Enterprise plans offer custom rate limiting options.
